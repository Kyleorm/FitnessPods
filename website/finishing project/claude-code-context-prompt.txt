FitnessPod Website - Project Context & Next Steps

I need you to fully understand where this project is at right now before we do anything else. Please read this carefully.

BACKGROUND
I am building a website for a client called FitnessPod IOM, a private gym on the Isle of Man with 6 individual workout pods. The website has already been built with branding, pages, and a custom designed booking grid UI (showing pods as columns and time slots as rows).

WHAT HAS CHANGED
Originally the plan was to build a fully custom booking system with Pod Points (a loyalty currency), gift cards, and our own payment + door code integration. The client has now decided NOT to go ahead with that. They want to keep using their existing booking system, which is called ClubSolution (run by a company called Globus Data).

I have already removed the Pod Points and gift card features from the website per the client's request. The custom booking grid UI design itself should remain - the client wants to keep that visual design and layout.

WHAT NEEDS TO HAPPEN NOW
ClubSolution has an API that I am getting access to. I confirmed directly with their support (a contact named Brian) what this API can and cannot do:

WHAT THE API CAN DO:
- Retrieve booking/court data and class data
- Allow me to build a visible UI showing availability (using my own design)
- Allow linking through to the ClubSolution booking pages

WHAT THE API CANNOT DO:
- No external user login through my website
- No creating bookings externally (no POST data)
- No class booking or membership purchases through my website
- All actual booking, payment and login logic MUST happen on ClubSolution's own pages

SO THE FINAL FLOW WILL BE:
1. Customer visits my custom designed website
2. Customer sees the booking grid (my design) showing live pod availability pulled from the ClubSolution API
3. Customer clicks an available slot
4. Customer is redirected to ClubSolution's own booking page to log in, complete the booking and pay (this happens on ClubSolution's site, not mine - their payment processor is called SecurePay360)
